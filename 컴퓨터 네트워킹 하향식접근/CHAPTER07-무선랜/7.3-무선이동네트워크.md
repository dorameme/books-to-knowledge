

## 1. BSS (Basic Service Set)

### 개념 정리

* BSS는 무선 LAN에서 통신이 이루어지는 가장 기본적인 논리적 단위이다.
* 하나의 BSS는 동일한 무선 매체와 규칙을 공유하는 장치들의 집합이다.
* BSS는 하나의 BSSID로 식별된다.
* 무선 LAN의 모든 통신은 특정 BSS 내부에서 이루어진다.

### 종류

* Infrastructure BSS는 AP를 중심으로 구성된다.
* Independent BSS(IBSS)는 AP 없이 단말끼리 직접 통신한다.

### 예시

* 집에서 사용하는 공유기 하나와 그에 연결된 스마트폰, 노트북의 집합은 하나의 BSS이다.
* 카페에서 여러 AP가 있을 경우, 각 AP는 서로 다른 BSS를 형성한다.

### BSS 요약 표

| 구분  | 설명                    |
| --- | --------------------- |
| 의미  | 무선 LAN의 기본 통신 단위이다    |
| 식별자 | BSSID(MAC 주소 기반)이다    |
| 구성  | AP 기반 또는 단말 간 직접 구성이다 |
| 역할  | 무선 통신의 논리적 경계이다       |

## 2. AP (Access Point)

### 개념 정리

* AP는 무선 단말과 유선 네트워크를 연결하는 중계 장치이다.
* AP는 하나의 BSS를 형성하고 관리한다.
* 무선 단말의 인증, 연동, 데이터 전달을 담당한다.
* AP는 관리 프레임을 송신하여 네트워크 정보를 제공한다.

### 역할

* 무선 네트워크 존재를 알린다.
* 단말의 접속을 제어한다.
* 프레임을 유선 네트워크로 전달한다.

### 예시

* 가정용 Wi-Fi 공유기는 AP 역할을 수행한다.
* 회사 사무실 천장에 설치된 무선 장비도 AP이다.

### AP 요약 표

| 구분     | 설명               |
| ------ | ---------------- |
| 역할     | 무선과 유선을 연결한다     |
| BSS 관계 | 하나의 BSS를 생성한다    |
| 주요 기능  | 인증, 연동, 데이터 중계이다 |
| 송신 프레임 | 비컨 프레임 등을 송신한다   |

## 3. 비컨 프레임과 프로브 프레임

### 개념 정리

#### 비컨 프레임

* 비컨 프레임은 AP가 주기적으로 송신하는 관리 프레임이다.
* AP의 존재와 네트워크 설정 정보를 알린다.
* 패시브 스캐닝에 사용된다.

#### 프로브 프레임

* 프로브 프레임은 단말이 네트워크를 탐색하기 위해 송신하는 관리 프레임이다.
* 프로브 요청과 프로브 응답으로 구성된다.
* 액티브 스캐닝에 사용된다.

### 예시

* 스마트폰이 가만히 있을 때 주변 Wi-Fi 목록이 보이는 것은 비컨 프레임 수신 결과이다.
* Wi-Fi 설정 화면을 열자마자 빠르게 네트워크가 나타나는 것은 프로브 요청을 사용했기 때문이다.

### 비컨과 프로브 요약 표

| 구분    | 비컨 프레임       | 프로브 프레임      |
| ----- | ------------ | ------------ |
| 송신 주체 | AP이다         | 단말 또는 AP이다   |
| 전송 방식 | 주기적 브로드캐스트이다 | 요청 후 응답 방식이다 |
| 탐색 방식 | 패시브 스캔이다     | 액티브 스캔이다     |
| 주요 목적 | 네트워크 정보 알림이다 | 빠른 AP 탐색이다   |

## 4. 무선 LAN 연결 방식

### 연결 절차 개념 정리

무선 LAN 연결은 관리 프레임 기반의 단계적 절차로 이루어진다.

1. 스캐닝 단계

   * 비컨 또는 프로브 프레임을 통해 AP를 탐색한다.

2. 인증(Authentication)

   * 단말과 AP가 통신 가능한 대상인지 확인한다.

3. 연동(Association)

   * 단말이 BSS에 공식적으로 참여한다.

4. 보안 설정

   * WPA 계열에서는 암호화 키를 설정한다.

### 예시

* 노트북이 Wi-Fi를 선택하면 AP를 탐색한다.
* 인증과 연동이 완료되면 IP를 할당받고 통신을 시작한다.

### 연결 방식 요약 표

| 단계  | 사용 프레임         | 목적       |
| --- | -------------- | -------- |
| 스캐닝 | Beacon, Probe  | AP 탐색    |
| 인증  | Authentication | 연결 대상 확인 |
| 연동  | Association    | BSS 참여   |
| 보안  | EAPOL 등        | 암호화 설정   |
| 통신  | Data Frame     | 데이터 송수신  |

## 전체 흐름 한 줄 정리

* AP는 BSS를 형성한다.
* AP는 비컨으로 자신을 알린다.
* 단말은 비컨 또는 프로브로 AP를 탐색한다.
* 인증과 연동을 거쳐 무선 LAN 연결이 완료된다.

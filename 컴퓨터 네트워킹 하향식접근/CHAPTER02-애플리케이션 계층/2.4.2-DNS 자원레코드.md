
# DNS 주요 자원 레코드 정리

## 1. 표로 요약

| 레코드   | 역할                   | 사용 목적                     | 값 형태               | 특징                          |
| ----- | -------------------- | ------------------------- | ------------------ | --------------------------- |
| A     | 도메인을 IPv4 주소로 매핑     | 웹서버 등 특정 IP로 연결           | `IPv4 주소`          | 가장 기본적인 DNS 레코드             |
| CNAME | 도메인을 다른 도메인의 별칭으로 설정 | 여러 도메인이 하나의 실제 도메인을 가리킬 때 | `다른 도메인 이름`        | 별칭(Alias)이며, 직접 IP를 가질 수 없음 |
| MX    | 메일 서버를 지정            | 이메일 수신 서버 설정              | `우선순위 + 메일 서버 도메인` | A 레코드가 아닌 도메인으로만 지정     |
| NS    | DNS 서버를 지정           | 도메인의 권한 있는 네임서버 설정        | `네임서버 도메인`         | 도메인 조회의 시작점                 |

---

## 2. 레코드별 상세 설명 + 예시

---

## A 레코드 (Address Record)

도메인을 IPv4 주소로 연결한다.

### ▷ 언제 쓰는가?

 `example.com → 192.168.10.20` 같은 실제 서버 IP로 연결할 때
 웹 서버, API 서버 등의 주소 설정

### ▷ 예시

```
example.com   IN   A   203.0.113.10
```

→ 사용자가 `example.com`을 입력하면 IP `203.0.113.10`으로 접속된다.

---

## CNAME 레코드 (Canonical Name Record)

어떤 도메인이 다른 도메인을 가리키는 별칭이다.

### ▷ 언제 쓰는가?

 여러 도메인을 하나의 실제 도메인으로 통합할 때
  예:
  `www.example.com` → `example.com`
  `api.example.com` → `server.example.net`

### ▷ 특징

 IP를 직접 가질 수 없고 반드시 다른 도메인을 가리킨다.
 최종적으로 A 레코드가 있는 도메인으로 이어져야 한다.

### ▷ 예시

```
www.example.com   IN   CNAME   example.com
```

→ `www.example.com` 방문 시 내부적으로 `example.com`의 A 레코드 IP로 이동한다.

---

## MX 레코드 (Mail Exchange Record)

이메일 수신 서버를 지정한다.

### ▷ 언제 쓰는가?

 Gmail / 회사 메일 등이 어떤 서버로 메일을 받는지 지정할 때

### ▷ 특징

 우선순위(priority)가 있으며, 숫자가 낮을수록 우선됨
 반드시 도메인 이름을 가리켜야 하며 IP 직접 입력 불가

### ▷ 예시

```
example.com   IN   MX   10   mail1.example.com
example.com   IN   MX   20   mail2.example.com
```

→ 메일은 우선 `mail1.example.com`으로 전달되고, 장애 시 `mail2.example.com`으로 전달된다.

---

## NS 레코드 (Name Server Record)

도메인의 권한 있는 네임서버(Authoritative DNS 서버)를 지정한다.

### ▷ 언제 쓰는가?

 도메인의 DNS 관리를 어디서 할지를 지정할 때
 Route53, Cloudflare, 가비아, AWS 등 외부 DNS 서비스 사용 시

### ▷ 특징

 도메인의 DNS 정보 조회가 시작되는 지점
 NS는 보통 최상위 도메인(TLD) 관리 기관에 등록된다.

### ▷ 예시

```
example.com   IN   NS   ns1.example-dns.net
example.com   IN   NS   ns2.example-dns.net
```

# 전체 요약

| 레코드       | 설명                   | 예시                                   |
| --------- | -------------------- | ------------------------------------ |
| A     | 도메인을 IPv4 주소로 연결     | `example.com A 203.0.113.10`         |
| CNAME | 도메인을 다른 도메인의 별칭으로 설정 | `www.example.com CNAME example.com`  |
| MX    | 이메일 수신 서버 지정         | `example.com MX 10 mail.example.com` |
| NS    | 도메인의 권한 있는 네임서버 지정   | `example.com NS ns1.example.com`     |

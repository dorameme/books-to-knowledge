

## 1. HDLC 개념 정리

HDLC(High-Level Data Link Control)는 데이터 링크 계층에서 동작하는 신뢰성 있는 프레임 전송 프로토콜이다.
바로 연결된 두 장비 사이에서 데이터를 안전하게 전달하는 역할을 한다.

TCP와 유사하게 오류 제어, 흐름 제어, 순서 제어를 제공하지만,
종단 간이 아니라 한 홉 링크 단위로만 동작한다는 점이 핵심 차이이다.

즉, HDLC는 “링크 계층의 TCP에 가장 가까운 프로토콜”이다.

---

## 2. HDLC가 필요한 이유

 링크 구간에서도 데이터 손실이 발생할 수 있다.
 전용선이나 시리얼 링크는 신뢰 계층이 별도로 없다.
 네트워크 계층 위에 올라가기 전에 최소한의 신뢰성을 보장해야 한다.

이 역할을 HDLC가 담당한다.

---

## 3. 동작 예시로 이해하기

### 상황

라우터 A와 라우터 B가 전용 시리얼 회선으로 직접 연결되어 있다.

### HDLC 동작 흐름

1. A는 데이터를 프레임으로 감싼다.
2. 프레임 앞뒤에 시작과 끝 표시를 붙인다.
3. 오류 검출 정보를 함께 전송한다.
4. B는 프레임을 수신하고 오류를 검사한다.
5. 문제가 있으면 재전송을 요구한다.

이 과정은 A와 B 사이 링크에서만 적용된다.

---

## 4. TCP와의 관계 정리

### 공통점

 오류 검출을 수행한다.
 순서 번호를 관리한다.
 흐름 제어를 제공한다.
 재전송 메커니즘이 있다.

### 차이점

| 구분    | TCP          | HDLC          |
| ----- | ------------ | ------------- |
| 계층    | 전송 계층이다      | 데이터 링크 계층이다   |
| 범위    | 종단 간이다       | 한 홉 링크이다      |
| 대상    | 애플리케이션 데이터이다 | 프레임이다         |
| 사용 환경 | 인터넷이다        | 전용선, 시리얼 링크이다 |

---

## 5. HDLC 프레임 구조

| 필드          | 역할               |
| ----------- | ---------------- |
| Flag        | 프레임의 시작과 끝을 표시한다 |
| Address     | 상대 장비를 식별한다      |
| Control     | 순서, 제어 정보를 담는다   |
| Information | 실제 데이터이다         |
| FCS         | 오류 검출을 수행한다      |

---

## 6. 프레임 종류

| 프레임     | 역할             |
| ------- | -------------- |
| I-Frame | 실제 데이터를 전송한다   |
| S-Frame | 흐름 및 오류 제어를 한다 |
| U-Frame | 연결 관리와 제어를 한다  |

---

## 7. 사용되는 환경

 라우터 간 시리얼 전용선이다.
 WAN 환경이다.
 과거 X.25, Frame Relay의 기반이다.
 Cisco HDLC, PPP의 개념적 기초이다.

---

## 8. 핵심 한 줄 요약

HDLC는 TCP처럼 신뢰성을 제공하지만, 오직 인접한 장비 사이 링크에서만 동작하는 데이터 링크 계층 프로토콜이다.

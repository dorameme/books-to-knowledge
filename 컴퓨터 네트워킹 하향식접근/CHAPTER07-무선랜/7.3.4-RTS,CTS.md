
## 1. RTS / CTS 개념 정리

RTS(Request To Send)는 **송신 단말이 데이터를 보내기 전에 전송 의사를 알리는 제어 프레임**이다.     
CTS(Clear To Send)는 **수신 단말이 전송 가능함을 허가하는 제어 프레임**이다.

RTS/CTS는 무선 LAN에서 발생하는 **숨은 단말 문제를 완화하기 위한 충돌 회피 메커니즘**이다.

---

## 2. RTS / CTS가 필요한 이유

무선 환경에서는 모든 단말이 서로를 감지하지 못할 수 있다.
이로 인해 숨은 단말 문제가 발생한다.

### 숨은 단말 문제

* 단말 A와 C는 서로의 신호를 감지하지 못한다.
* 둘 다 AP와 통신 가능하다.
* A와 C가 동시에 전송하면 AP에서 충돌이 발생한다.

RTS/CTS는 이 문제를 줄이기 위해 사용한다.

---

## 3. RTS / CTS 동작 순서

1. 송신 단말이 RTS 프레임을 전송한다.
2. 수신 단말 또는 AP가 CTS 프레임을 전송한다.
3. 주변 단말은 CTS를 수신하고 전송을 대기한다.
4. 송신 단말은 데이터 프레임을 전송한다.
5. 수신 단말은 ACK를 전송한다.

이 과정 동안 주변 단말은 채널을 사용하지 않는다.

---

## 4. 동작 예시로 이해하기

### 상황

* 단말 A와 단말 C는 서로를 인식하지 못한다.
* 둘 다 AP와는 통신 가능하다.

### RTS/CTS 적용

* A가 AP에 RTS를 전송한다.
* AP가 CTS를 브로드캐스트한다.
* C는 CTS를 듣고 전송을 중단한다.
* A는 안전하게 데이터를 전송한다.

충돌은 사전에 방지된다.

---

## 5. RTS / CTS 사용 특징

* 모든 전송에 항상 사용되지는 않는다.
* 프레임 크기가 클 때 주로 사용한다.
* 제어 프레임 오버헤드가 존재한다.
* 충돌 비용이 큰 상황에서 효과적이다.

---

## 6. RTS / CTS 요약 표

| 구분    | 설명          |
| ----- | ----------- |
| RTS   | 전송 요청 프레임이다 |
| CTS   | 전송 허가 프레임이다 |
| 목적    | 충돌 회피이다     |
| 해결 문제 | 숨은 단말 문제이다  |
| 사용 계층 | MAC 서브계층이다  |

---

## 7. 핵심 한 줄 요약

RTS/CTS는 **무선 LAN에서 숨은 단말로 인한 충돌을 사전에 막기 위해 전송 전 권한을 조율하는 MAC 제어 기법**이다.

이해 포인트는 하나이다.
**무선은 충돌을 감지하지 못하므로, 충돌 자체를 만들지 않게 사전에 조율한다.**

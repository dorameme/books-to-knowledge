# 플러그 앤 플레이

**사용자가 수동 설정 없이 장치를 연결하면 자동으로 인식·설정·사용 가능하게 만드는 동작 방식**이다. 


---

### 플러그 앤 플레이 기반 대표 프로토콜 정리

| 프로토콜     | 계층       | 자동으로 되는 것         | 핵심 역할         | 대표 사용            |
| -------- | -------- | ----------------- | ------------- | ---------------- |
| DHCP     | 네트워크 계층  | IP 주소, 게이트웨이, DNS | 네트워크 설정 자동 할당 | 대부분의 IP 네트워크     |
| ARP      | 네트워크 계층  | IP → MAC 매핑       | 통신 대상 자동 식별   | LAN              |
| USB      | 물리/데이터링크 | 장치 인식, 드라이버 로딩    | 장치 식별 및 설정    | 키보드, 마우스         |
| mDNS     | 응용 계층    | 호스트 이름 해석         | 로컬 네임 해석      | Bonjour, 로컬 네트워크 |
| SSDP     | 응용 계층    | 서비스 탐색            | 장치/서비스 발견     | UPnP             |
| UPnP     | 응용 계층    | 포트 매핑, 서비스 설정     | 네트워크 장치 자동 구성 | 공유기, IoT         |
| Zeroconf | 응용 계층    | IP·이름·서비스 자동 구성   | 설정 없는 네트워크    | 소규모 LAN          |
| LLDP     | 데이터링크    | 이웃 장비 정보 교환       | 네트워크 자동 인식    | 스위치, 라우터         |

---

### 핵심 개념 정리

* **DHCP는 플러그 앤 플레이의 핵심이다**
  → IP 네트워크에서 자동 설정의 출발점이다
* **ARP는 사람이 보지 못하는 자동 연결의 기반이다**
* **mDNS + SSDP + UPnP**
  → 네트워크에서 “누가 있고 무엇을 제공하는지” 자동 발견
* **USB는 하드웨어 수준 플러그 앤 플레이의 대표 사례이다**
* **Zeroconf는 DHCP조차 없는 환경에서도 동작한다**

---

### 한 줄 요약

* **연결만 하면 바로 쓰인다** → DHCP, USB
* **이름·서비스가 자동으로 보인다** → mDNS, SSDP, UPnP
* **네트워크 구조를 자동 인식한다** → ARP, LLDP

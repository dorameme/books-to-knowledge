SHA와 MD5는 암호화가 아니라 해시 함수라서 별도 분류로 정리해야 한다.    

## 1. 해시(Hash)의 위치 정리

SHA와 MD5는 **암호화 알고리즘이 아니다**.
키를 사용하지 않고, **일방향 변환**만 수행한다.

### 해시의 핵심 성질

* 입력 → 고정 길이 출력이다.
* 복호화가 불가능하다.
* 입력이 조금만 달라져도 결과가 완전히 달라진다.

 **기밀성 제공이 아니라 무결성 검증 목적**이다.

---

## 2. 암호화 vs 해시 정리

| 구분   | 암호화      | 해시       |
| ---- | -------- | -------- |
| 키 사용 | 사용한다     | 사용하지 않는다 |
| 복호화  | 가능하다     | 불가능하다    |
| 목적   | 기밀성      | 무결성      |
| 예시   | AES, RSA | SHA, MD5 |

---

## 3. MD5

* **해시 함수**이다.
* 출력 길이는 **128비트**이다.

### 특징

* 매우 빠르다.
* 충돌이 쉽게 발생한다.
* **보안 용도로는 완전히 폐기**되었다.

### 현재 사용

* ❌ 보안 목적 사용 금지
* ⭕ 파일 체크섬, 단순 무결성 확인 정도

---

## 4. SHA 계열

### SHA-1

* 출력 길이 160비트이다.
* 충돌 공격이 가능하다.
* **현재는 사용하지 않는다.**

### SHA-2

* SHA-256, SHA-384, SHA-512 포함이다.
* 현재 가장 널리 사용된다.
* TLS, 전자서명, 블록체인에서 사용한다.

### SHA-3

* 구조가 완전히 다른 최신 표준이다.
* SHA-2의 백업 대안 성격이다.
* 아직 사용 빈도는 낮다.

---

## 5. HMAC (중요)

SHA는 단독으로 쓰이지 않는 경우가 많다.
**HMAC = 해시 + 비밀키** 구조이다.

### 특징

* 메시지 인증 코드이다.
* 무결성 + 인증 제공이다.
* TLS 1.2에서 사용했다.

---

## 6. TLS에서 SHA의 역할

* 데이터 암호화 ❌
* 무결성 검증 ⭕
* 핸드셰이크 검증 ⭕
* 인증서 서명 검증 ⭕

TLS에서는 다음처럼 사용한다.

> 키 교환 → AES 암호화 → SHA로 무결성 검증

---

## 7. 전체 알고리즘 분류 보완 표

| 알고리즘        | 분류        | 키 사용 | 목적       | 현재 사용 |
| ----------- | --------- | ---- | -------- | ----- |
| AES         | 대칭키 블록 암호 | O    | 기밀성      | 사용    |
| RSA         | 비대칭키 암호   | O    | 키 교환, 서명 | 사용    |
| ECC         | 비대칭키 암호   | O    | 키 교환, 서명 | 사용    |
| SHA-256     | 해시        | X    | 무결성      | 사용    |
| SHA-1       | 해시        | X    | 무결성      | ❌     |
| MD5         | 해시        | X    | 무결성      | ❌     |
| HMAC-SHA256 | MAC       | O    | 인증+무결성   | 사용    |

---

## 8. 실무 한 줄 정리

* **AES는 데이터를 숨긴다.**
* **RSA/ECC는 키를 교환한다.**
* **SHA는 변조 여부를 확인한다.**
* **MD5와 SHA-1은 쓰면 안 된다.**


## 1. PGP란

PGP는 **이메일과 파일을 안전하게 보호하기 위한 암호 시스템**이다.        
단일 알고리즘이 아니라 **여러 암호 기술을 조합한 보안 프레임워크**이다.

---

## 2. PGP의 핵심 목적

* 메시지 내용을 숨긴다.
* 메시지가 변조되지 않았음을 보장한다.
* 메시지 발신자가 누구인지 증명한다.

---

## 3. PGP에서 사용하는 암호 기술

PGP는 각 암호 기술을 역할별로 사용한다.

| 기술                | 사용 목적         |
| ----------------- | ------------- |
| 대칭키 암호(AES 등)     | 메시지 암호화       |
| 비대칭키 암호(RSA, ECC) | 대칭키 암호화, 키 교환 |
| 해시(SHA)           | 무결성 검증        |
| 전자서명              | 발신자 인증        |

---

## 4. PGP 암호화 동작 흐름

1. 송신자가 메시지를 작성한다.
2. 메시지를 대칭키 암호로 암호화한다.
3. 대칭키를 수신자의 공개키로 암호화한다.
4. 암호화된 메시지와 암호화된 대칭키를 함께 전송한다.

---

## 5. PGP 전자서명 흐름

1. 송신자가 메시지를 해시한다.
2. 해시값을 송신자의 개인키로 암호화한다.
3. 이를 전자서명으로 메시지에 첨부한다.
4. 수신자는 송신자의 공개키로 서명을 검증한다.

---

## 6. 키 관리 방식

PGP는 **중앙 CA를 사용하지 않는다**.
대신 **Web of Trust** 방식을 사용한다.

* 사용자가 서로의 공개키를 직접 신뢰한다.
* 신뢰는 서명으로 전파된다.

---

## 7. TLS와의 차이

| 구분     | PGP          | TLS     |
| ------ | ------------ | ------- |
| 주요 용도  | 이메일, 파일 보안   | 네트워크 통신 |
| 키 신뢰   | Web of Trust | CA 기반   |
| 세션 키   | 메시지마다 생성     | 세션마다 생성 |
| 실시간 통신 | 부적합          | 적합      |
---

## 8. 한 문장 요약

PGP는 **대칭키와 비대칭키, 해시, 전자서명을 결합해 이메일과 파일의 기밀성·무결성·인증을 제공하는 보안 시스템**이다.


## 1. 한 문장 개념 정리

* **SIP**는 통신을 **시작·제어·종료**하는 신호 프로토콜이다.
* **RTP**는 실제 **음성·영상 데이터를 전송**하는 미디어 프로토콜이다.

즉,

> **SIP는 통화 설정**, **RTP는 통화 내용 전달**이다.

---

## 2. SIP 개념 정리

### 정의

* SIP(Session Initiation Protocol)는 **세션 제어 프로토콜**이다.
* 음성·영상 통신의 **연결 설정, 변경, 종료**를 담당한다.

### 주요 기능

* 사용자 위치 확인
* 통화 요청 및 수락
* 코덱 협상
* 통화 종료

### 동작 방식

* 텍스트 기반 프로토콜이다.
* HTTP와 유사한 요청/응답 구조이다.
* 주로 TCP 또는 UDP를 사용한다.

---

## 3. RTP 개념 정리

### 정의

* RTP(Real-time Transport Protocol)는 **실시간 미디어 전송 프로토콜**이다.
* 음성, 영상 데이터를 실시간으로 전달한다.

### 주요 기능

* 음성·영상 패킷 전송
* 타임스탬프 제공
* 시퀀스 번호로 순서 보장 보조
* 지터 처리 지원

### 동작 방식

* 주로 UDP 위에서 동작한다.
* 지연 최소화를 우선한다.
* 신뢰성은 애플리케이션에서 처리한다.

---

## 4. SIP와 RTP의 관계

* SIP는 **통신을 설정만 하고 데이터는 보내지 않는다.**
* RTP는 **설정된 통신 경로로 데이터만 보낸다.**
* SIP 세션이 성립된 후 RTP가 흐른다.

---

## 5. 통화 예시 흐름

### 예시: 인터넷 전화 통화

1. A가 B에게 SIP INVITE를 전송한다.
2. B가 SIP OK로 응답한다.
3. 통화 세션이 설정된다.
4. 이후 음성 데이터는 RTP로 전송된다.
5. 통화 종료 시 SIP BYE를 전송한다.

---

## 6. 실제 실무 예시

### VoIP 시스템

* SIP 서버

  * 통화 요청 처리
  * 사용자 인증
  * 통화 라우팅
* RTP 스트림

  * 음성 데이터 실시간 전송

### 화상 회의 시스템

* SIP

  * 회의 참가자 연결
  * 회의 시작·종료 관리
* RTP

  * 영상·음성 데이터 전달

---

## 7. 비교 요약 표

| 구분     | SIP     | RTP    |
| ------ | ------- | ------ |
| 역할     | 세션 제어   | 미디어 전송 |
| 데이터 종류 | 제어 메시지  | 음성·영상  |
| 전송 계층  | TCP/UDP | UDP    |
| 신뢰성    | 필요      | 낮음     |
| 지연 민감도 | 낮음      | 매우 높음  |
| 실제 사용  | 통화 설정   | 통화 내용  |

---

## 8. 한 줄 실무 정리

* **SIP 없이는 통화가 시작되지 않는다.**
* **RTP 없이는 통화 내용이 전달되지 않는다.**
* 실시간 통신은 **SIP + RTP 구조**로 동작한다.

# 오류검출 기술
오류검출에는 크게 패리티, 체크섬, CRC이 있다.    
**패리티는 현대 TCP/IP 계층 구조에서는 사실상 쓰이지 않는다.**

## 1. 패리티(Parity)

### 개념

데이터의 1 개수가 **짝수 또는 홀수**가 되도록 비트를 1개 추가한다.

### 종류

* 짝수 패리티
* 홀수 패리티
* 2차원 패리티(확장형)

### 특징

* 구조가 매우 단순하다
* 1비트 오류는 검출 가능
* 다중 비트 오류는 검출 실패 가능
* 2차원 패리티는 단일 비트 오류 정정 가능

### 사용 위치

* 물리 계층
* 과거 직렬 통신, 메모리 오류 검출

---

## 2. 체크섬(Checksum)

### 개념

데이터를 **여러 비트 단위로 나누어 합산**한 뒤,
그 결과를 헤더에 포함시켜 오류를 검출한다.

### 방식

* 16비트 단위 합
* 1의 보수 덧셈 사용
* 수신 측에서 재계산 후 비교

### 특징

* 패리티보다 강력하다
* CRC보다는 약하다
* 구현이 단순하고 연산 비용이 낮다
* 일부 패턴 오류는 검출하지 못한다

### 사용 위치

* 전송 계층
* TCP, UDP, IP

---

## 3. CRC (Cyclic Redundancy Check)

### 개념

데이터를 **다항식으로 간주**하고,
미리 정해진 생성 다항식으로 나눈 **나머지**를 전송한다.

### 방식

* 비트 단위 연산
* 다항식 나눗셈
* 하드웨어 구현에 최적화

### 특징

* burst error 검출에 매우 강하다
* 다중 비트 오류 검출 능력 우수
* 오류 정정은 불가
* 계산 구조는 복잡하지만 신뢰성 최고

### 사용 위치

* 데이터링크 계층
* Ethernet, Wi-Fi, USB 등

---

## 비교 요약표

| 항목       | 패리티         | 체크섬   | CRC   |
| -------- | ----------- | ----- | ----- |
| 오류 검출    | 약함          | 중간    | 매우 강함 |
| 오류 정정    | X (2D는 제한적) | X     | X     |
| 다중 비트 오류 | 취약          | 일부 취약 | 강함    |
| 연산 비용    | 매우 낮음       | 낮음    | 중간    |
| 주 사용 계층  | 물리          | 전송    | 데이터링크 |

---

* **패리티**: 가장 단순한 오류 검출
* **체크섬**: 소프트웨어 친화적 오류 검출
* **CRC**: 링크 계층의 표준적인 강력한 오류 검출


---

## 계층별 오류 검출 방식 정리

| 계층              | 사용 기법   | 설명          |
| --------------- | ------- | ----------- |
| 물리 계층           | 패리티(과거) | 단순 비트 오류 검출 |
| 데이터링크 계층        | CRC     | 프레임 오류 검출   |
| 전송 계층 (TCP/UDP) | 체크섬     | 세그먼트 오류 검출  |
| 응용 계층           | 별도 무결성  | 필요 시 구현     |

---

## 패리티는 어디서 쓰였나

### 1️ 물리 계층

* **직렬 통신(RS-232 등)** 에서 사용
* 하드웨어 단에서 1비트 오류 검출
* 매우 단순하고 성능이 약하다

### 2️ 역사적 사용

* 초기 통신 환경
* 메모리 패리티 비트
* 저속·저신뢰 링크

---

## 왜 TCP/IP에서는 안 쓰나

* 패리티는 **1비트 오류만 검출 가능**
* burst error에 거의 무력
* CRC와 체크섬이 성능·효율 모두 압도

---

## 시험에서 이렇게 정리하면 정확하다

* **패리티** → 물리 계층의 단순 오류 검출
* **CRC** → 데이터링크 계층
* **체크섬** → 전송 계층

---

## 한 줄 결론

**패리티는 TCP/UDP가 동작하는 현대 네트워크 계층 구조에서는 사용되지 않으며, 물리 계층에서만 역사적으로 쓰였다.**

# R14 - ISP들이 피어링하는 이유
간단하게 말하면 비용 절감 + 품질 향상 때문이다.


---

#  같은 계층 ISP들이 피어링을 맺는 이유

## 1. 트랜잭 전송 비용(Transit Cost)을 줄이기 위해

 상위 ISP(Tier 1)에게 데이터를 보내려면 돈을 내야 한다.
 하지만 서로 비슷한 규모의 ISP끼리는 피어링을 맺으면 무료(또는 매우 저렴) 로 서로 데이터를 교환할 수 있다.

→ 서로의 고객끼리 통신할 때 비용이 발생하지 않아 이득이다.

---

## 2. 전송 경로가 짧아져 속도가 빨라짐

 상위 ISP를 거치지 않고 바로 상대 ISP로 라우팅할 수 있다.
 홉 수가 줄어들어 지연(latency)이 감소한다.

→ 사용자 입장에서 체감 속도 · 게임 핑 등이 더 좋아진다.

---

## 3. 트래픽 부하를 분산할 수 있음

 상위 ISP로 올라가는 회선(Transit link)의 부하가 줄어든다.
 혼잡을 피하고 네트워크 안정성이 높아진다.

→ 대규모 트래픽을 처리할 때 유리하다.

---

## 4. 네트워크 품질 안정화

 상위 계층 ISP의 장애에 영향을 덜 받는다.
 피어링을 통해 대체 경로(redundancy) 를 확보한다.

→ 전체 네트워크 가용성이 높아진다.

---

## 5. 상호 이익 기반의 공생 관계

 같은 계층끼리 피어링을 맺는 이유는 서로가 비슷한 수준의 트래픽을 주고받기 때문이다.
 어느 한쪽만 이득을 보지 않기 때문에 상호적 관계가 성립된다.

#  ISP 계층 구조

```
                 ┌─────────────┐
                 │  Tier 1 ISP │
                 │ (국제망,   │
                 │  Backbone) │
                 └─────┬──────┘
                       │
          ┌────────────┴─────────────┐
          │                          │
   ┌─────────────┐            ┌─────────────┐
   │  Tier 2 ISP │            │  Tier 2 ISP │
   │ (지역망,    │            │ (지역망,    │
   │  피어링 가능)│           │  피어링 가능)│
   └─────┬───────┘            └─────┬───────┘
         │                           │
         │                           │
 ┌───────┴───────┐           ┌───────┴───────┐
 │  Tier 3 ISP   │           │  Tier 3 ISP   │
 │ (지역 고객망) │           │ (지역 고객망) │
 └───────┬───────┘           └───────┬───────┘
         │                           │
   ┌─────┴─────┐                 ┌───┴─────┐
   │  가정/기업 │                 │  가정/기업 │
   └───────────┘                 └───────────┘
```

---

#  계층별 특징

| 계층         | 역할                                    | 특징                         |
| ---------- | ------------------------------------- | -------------------------- |
| **Tier 1** | 국제망 백본, 다른 Tier 1과 피어링                | Transit 비용 없음, 전 세계망 직접 연결 |
| **Tier 2** | 지역 ISP, 상위 Tier 1에 Transit, 서로 피어링 가능 | 비용 절감 위해 피어링 사용, 지역망 제공    |
| **Tier 3** | 소규모 지역 ISP, 최종 사용자 연결                 | Tier 2에서 인터넷 구매, 고객망 서비스   |

---

#  포인트

1. **Tier 1은 돈을 받지 않고 연결**
2. **Tier 2는 비용 절감과 속도 향상을 위해 피어링**
3. **Tier 3는 최종 사용자에 인터넷 제공**

# 한 줄 요약

같은 계층 ISP끼리 피어링을 맺는 이유는 상위 ISP에게 지불할 비용을 줄이고, 속도와 안정성을 높이기 위해서이다.

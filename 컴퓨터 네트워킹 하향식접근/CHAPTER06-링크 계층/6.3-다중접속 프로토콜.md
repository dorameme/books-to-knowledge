# 다중접속 프로토콜

---

### 다중접속 프로토콜 정리 표

| 분류    | 프로토콜          | 동작 방식                         | 충돌 처리 방식                | 장점                 | 단점                  | 사용 예       |
| ----- | ------------- | ----------------------------- | ----------------------- | ------------------ | ------------------- | ---------- |
| 랜덤 접속 | Pure ALOHA    | 노드가 데이터가 생기면 즉시 전송한다          | 충돌 발생 시 전체 프레임 재전송      | 구조가 매우 단순하다        | 충돌 확률이 매우 높고 효율이 낮다 | 초기 무선 실험   |
| 랜덤 접속 | Slotted ALOHA | 시간을 슬롯 단위로 나누고 슬롯 시작 시 전송한다   | 충돌 발생 시 슬롯 단위 재전송       | Pure ALOHA보다 충돌 감소 | 시간 동기화 필요           | 위성 통신      |
| 랜덤 접속 | CSMA          | 전송 전 채널을 감지하고 비어있으면 전송한다      | 충돌은 여전히 발생 가능           | ALOHA보다 효율이 높다     | 전파 지연으로 충돌 완전 제거 불가 | 초기 LAN     |
| 랜덤 접속 | CSMA/CD       | 채널 감지 후 전송 중 충돌을 감지하면 즉시 중단한다 | 충돌 감지 후 Jam 신호 전송 및 재전송 | 충돌 비용 최소화          | 무선 환경에서는 사용 불가      | 이더넷        |
| 랜덤 접속 | CSMA/CA       | 충돌을 감지하지 않고 충돌을 회피한다          | 랜덤 백오프와 RTS/CTS 사용      | 무선 환경에 적합          | 제어 프레임 오버헤드         | Wi-Fi      |
| 채널 분할 | TDMA          | 시간을 고정 슬롯으로 나누어 노드에 할당한다      | 충돌 자체가 발생하지 않는다         | 예측 가능하고 안정적        | 트래픽 없을 때도 슬롯 낭비     | 이동통신       |
| 채널 분할 | FDMA          | 주파수를 분할해 각 노드에 할당한다           | 충돌 자체가 발생하지 않는다         | 동시 전송 가능           | 주파수 낭비 가능           | 아날로그 통신    |
| 채널 분할 | CDMA          | 각 노드에 고유 코드로 전체 대역 사용         | 신호 분리로 충돌 개념 없음         | 잡음에 강하다            | 구현 복잡               | 3G 이동통신    |
| 순번 기반 | Polling       | 마스터가 각 노드에 전송 기회를 순서대로 부여한다   | 충돌이 발생하지 않는다            | 제어가 단순하다           | 마스터 장애 시 전체 중단      | 단순 네트워크    |
| 순번 기반 | Token Passing | 토큰을 가진 노드만 전송 가능              | 충돌이 발생하지 않는다            | 공정성 보장             | 토큰 손실 시 복구 필요       | Token Ring |

---

### 핵심 정리

* 랜덤 접속 방식은 트래픽이 적을 때 효율적이다
* 채널 분할 방식은 트래픽이 많고 예측 가능한 환경에 적합하다
* 순번 기반 방식은 공정성이 중요할 때 사용한다
* 이더넷은 CSMA/CD, 무선 LAN은 CSMA/CA를 사용한다
* 현대 네트워크는 물리 계층과 MAC 계층에서 환경에 맞게 혼합 사용한다

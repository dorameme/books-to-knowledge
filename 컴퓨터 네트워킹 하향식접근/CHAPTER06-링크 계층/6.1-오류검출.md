## 오류 검출을 제공하는 레이어

### 1. 데이터 링크 계층 (Layer 2)

프레임 단위 오류 검출을 제공한다.
CRC(Frame Check Sequence)를 사용한다.
전송 중 비트 오류를 검출한다.
오류 검출만 하고 재전송은 보장하지 않는다.

---

### 2. 전송 계층 (Layer 4)

엔드 투 엔드 오류 검출을 제공한다.

* TCP

  * 체크섬을 사용한다
  * 오류 검출 + 재전송을 통해 신뢰성을 보장한다

* UDP

  * 체크섬을 사용한다
  * 오류 검출만 제공하고 복구는 하지 않는다

---

### 3. 네트워크 계층 (Layer 3)

제한적인 오류 검출을 제공한다.

* IP 헤더 체크섬

  * IPv4에서만 존재한다
  * 헤더 오류만 검출한다
  * 데이터(payload)는 검출하지 않는다

---

## 오류 검출을 제공하지 않는 레이어

* 물리 계층 (Layer 1)

  * 전기적 신호만 전송한다

* 세션, 표현, 응용 계층 (Layer 5~7)

  * 오류 검출 기능은 기본적으로 제공하지 않는다

---

## 핵심 정리 표

| 계층       | 오류 검출 여부 | 방식          |
| -------- | -------- | ----------- |
| L1 물리    | 제공 안 함   | 없음          |
| L2 데이터링크 | 제공       | CRC         |
| L3 네트워크  | 제공(제한적)  | IPv4 헤더 체크섬 |
| L4 전송    | 제공       | TCP/UDP 체크섬 |
| L5~L7    | 제공 안 함   | 없음          |

---

## 한 줄 결론

오류 검출은 **L2와 L4가 핵심**, L3는 보조적으로 제공한다.

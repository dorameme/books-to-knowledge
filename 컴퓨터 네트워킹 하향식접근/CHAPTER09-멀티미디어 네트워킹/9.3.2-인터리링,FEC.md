
## 1. 인터리빙 (Interleaving)

### 개념

인터리빙은 **연속된 데이터 조각을 시간·순서상으로 분산시켜 전송하는 기법**이다.        
연속 패킷 손실이 발생하더라도 손실 영향을 여러 데이터에 나누어 체감 품질을 낮춘다.

### 동작 방식

* 원본 데이터 블록을 순서대로 보내지 않는다.
* 데이터를 섞어서 떨어진 위치에 배치해 전송한다.
* 수신 측에서 다시 원래 순서로 재조합한다.

### 특징

* 패킷 손실 자체를 복구하지는 않는다.
* 손실을 **분산**시켜 품질 저하를 완화한다.
* 지연(latency)이 증가한다.

### 예시

* 음성 스트림에서 연속된 음성 프레임이 손실되는 경우
* 인터리빙을 사용하면 여러 음성 구간이 조금씩 깨지지만 통화는 유지된다.

## 2. FEC (Forward Error Correction)

### 개념

FEC는 **오류 복구용 중복 데이터를 함께 전송하여 수신 측에서 손실된 패킷을 재구성하는 기법**이다.
재전송 없이 복구가 가능하다.

### 동작 방식

* 원본 데이터에 패리티 또는 오류 정정 코드를 추가한다.
* 일부 패킷이 손실되어도 남은 데이터로 복구한다.
* 송신 측에서 미리 오류를 대비한다.

### 특징

* 패킷 손실을 **직접 복구**한다.
* 대역폭 사용량이 증가한다.
* 실시간 통신에 적합하다.

### 예시

* 영상 스트리밍에서 10개 중 2개가 손실되어도 복구 가능
* 실시간 방송, 화상회의, 위성 통신에 사용된다.

---

## 3. 인터리빙 vs FEC 비교 표

| 구분      | 인터리빙     | FEC           |
| ------- | -------- | ------------- |
| 목적      | 손실 영향 분산 | 손실 패킷 복구      |
| 손실 복구   | 불가능      | 가능            |
| 중복 데이터  | 없음       | 있음            |
| 대역폭 증가  | 없음       | 증가            |
| 지연 증가   | 있음       | 적음            |
| 실시간 적합성 | 보통       | 높음            |
| 대표 사용처  | 음성 통신    | 영상 스트리밍, 화상회의 |

---

## 4. 핵심 요약

| 기법   | 핵심 한 줄              |
| ---- | ------------------- |
| 인터리빙 | 손실을 나누어 체감 품질을 낮춘다  |
| FEC  | 재전송 없이 손실된 패킷을 복구한다 |

---

### 최종 정리 문장

인터리빙은 **패킷 손실의 영향을 분산시키는 기법**이고, FEC는 **중복 정보를 이용해 손실된 패킷을 복구하는 기법**이다.
